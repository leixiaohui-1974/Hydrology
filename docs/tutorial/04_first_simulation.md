# 第4章：你的第一个水文模拟

理论是重要的，但没有什么比亲手运行一次模拟更能带来直观的感受。在本章中，我们将运行项目中最基础的一个示例，它将模拟一个由多个子流域组成的完整流域的径流过程。

## 示例脚本

我们将要运行的脚本是：

```
examples/run_example.py
```

这个脚本是我们在项目初期构建的，它使用了我们最简单的产流和汇流模块，但完整地演示了准分布式模型的概念。

## 运行模拟

请打开您的命令行工具，确保您位于项目的根目录下，然后执行以下命令：

```bash
python examples/run_example.py
```

您应该会看到类似以下的输出：

```
模拟完成，结果已保存到 results/simulation_results.csv
对比数据表已保存到 results/final_comparison_table.csv
对比图已保存到 results/comparison_plot.png
```

这表明程序已经成功运行！

## 解读结果

现在，请打开 `examples/results/` 目录，您会看到几个新生成的文件。其中最直观的是 `comparison_plot.png`。让我们来看一下这张图：

*(我们在这里可以插入`comparison_plot.png`的图像，以供教学使用)*

```
# (此处的图像是之前生成的，仅作示意)
# examples/results/comparison_plot.png
```

这张图告诉我们什么？

-   **蓝色条状图 (顶部，倒置)**: 代表了在总出水口处的降雨时间序列。这是驱动整个水文过程的输入。
-   **黑色散点**: 代表了我们输入的“真实”的、在出水口观测到的流量数据。这是我们希望模型能够模拟的目标。
-   **红色虚线**: 这是我们模型的**模拟输出**。它代表了模型根据输入的降雨，计算出的在出水口的流量过程线。

通过对比红色虚线和黑色散点，我们可以直观地评估模型的表现。在这个基础示例中，您可以看到模型大致捕捉了流量随降雨而涨落的趋势，但在洪峰量级等方面与真实观测存在一定差距。

**这正是我们后续章节要解决的问题！** 如何通过替换更高级的算法、以及使用数据同化技术，来让红线更完美地拟合黑点。

## 背后发生了什么？

当您运行这个脚本时，程序在后台执行了以下操作：
1.  读取 `data/` 目录下的CSV文件，获取了流域的拓扑结构（哪个子流域汇入哪个）、每个子流域的参数以及气象数据。
2.  创建了一个 `Catchment` 对象，它代表了整个流域。
3.  程序根据拓扑关系，从最上游的子流域开始，依次计算每个子流域的产流和汇流。
4.  下游子流域会接收来自其所有上游子流域的汇流，并叠加上自己本地产生的径流，再进行演算。
5.  最终，程序得到了在总出水口处的流量过程线，并将其与观测数据一同绘制出来。
