# 第10章：挑战环状河网

到目前为止，我们处理的都是“树状”的河网拓扑：水流从上游到下游，方向是唯一的。但现实世界中的河网，尤其是在平原地区或三角洲，经常会出现**环路 (Loops)**，例如，一条河流分叉成两条支流，然后在下游又重新汇合。

对于这种环状河网，简单的拓扑排序方法失效了，因为水流的分配和汇合是相互影响的。为了解决这个问题，我们为`Catchment`类实现了一个更强大的求解器：**迭代求解器**。

## 迭代求解的原理

当网络中存在环路时，我们无法一次性计算出正确的流量。例如，在分叉点，水流应该如何分配到下游的两条支流中？这取决于两条支流的水力条件，而水力条件又反过来受流量的影响。这是一个“鸡生蛋，蛋生鸡”的问题。

迭代求解器的思想很简单：
1.  **猜测**: 先对环路中的流量做一个初始的猜测。
2.  **演算**: 基于这个猜测，对整个网络进行一次完整的汇流演算。
3.  **检查**: 演算完成后，检查在环路的汇合点，水量是否平衡？在分叉点，流量分配是否合理？
4.  **修正**: 如果不平衡或不合理，就根据当前的计算结果，对环路中的流量进行修正，然后回到第2步。
5.  **重复**: 不断重复“演算-检查-修正”这个过程，直到整个网络中每个节点的流量都不再有明显变化，即达到了**收敛**状态。

## `run_iterative_simulation` 方法

我们在`Catchment`类中实现了`run_iterative_simulation`方法来执行这个过程。

```python
# hydro_model/catchment.py

def run_iterative_simulation(self, ..., max_iter=20, convergence_threshold=0.01):
    # ...
    for t in range(num_steps): # 对每一个时间步
        # ...
        for i in range(max_iter): # 在时间步 t 内，开始迭代

            # ... 存储上一次迭代的结果 ...

            # ... 对网络中的所有河段进行一次完整演算 ...

            # ... 更新所有节点的出流量 ...

            # ... 检查新旧两次迭代结果的差异 ...
            if total_change < convergence_threshold:
                break # 如果差异足够小，则认为已收敛，跳出迭代
    # ...
```
这个方法的核心就是在一个时间步`t`内部，嵌套了一个`for i in range(max_iter)`循环。它会不断地对整个河网的流量进行演算和更新，直到所有节点的流量值都稳定下来。

为了防止无限循环，我们设置了 `max_iter` (最大迭代次数) 和 `convergence_threshold` (收敛阈值) 两个安全参数。

**脚本:** `examples/run_looped_network_example.py`

这个示例构建了一个包含分叉和合流的“菱形”环状河网，并使用`run_iterative_simulation`方法进行模拟。

```python
# examples/run_looped_network_example.py

# ... 构建一个包含R1到R6的菱形网络 ...

# 调用迭代求解器
reach_q, node_q = network.run_iterative_simulation(...)
```

运行该脚本，并查看生成的图表 `results/looped_network_plot.png`。

*(我们在这里可以插入`looped_network_plot.png`的图像)*

您可以看到：
-   上游的总入流（黑线）在分叉点被分配到了西侧分支（蓝线）和东侧分支（绿线）中。由于我们为东侧分支设置了更大的曼宁糙率（意味着阻力更大），所以西侧分支分配到了更多的流量。
-   在下游，两条分支的流量重新汇合，并经过最终河段的演算，形成了总出流（红线）。

这证明了我们的迭代求解器成功地处理了环状河网的复杂水力学计算。

## 总结

通过实现迭代求解器，我们的框架现在能够处理任意复杂的河网拓扑，包括平原地区常见的环状、网状河网，使其成为一个功能非常完备的河网水动力模拟工具。
