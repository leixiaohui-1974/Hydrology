# 第7章：汇流模块详解

汇流过程模拟的是产出的径流如何通过流域的河道网络，汇集到出口断面。这个过程通常会使洪水波发生变形，表现为**洪峰的削减**和**时间的延迟**。本章将介绍我们框架中集成的几种汇流算法。

## 1. `SimpleRouting`

**位置:** `hydro_model/routing.py`

这是从我们最初的简单概念模型中重构出的汇流模块。

### 核心思想

-   **线性水库**: 该模型将入流（来自产流模块的径流）分为快速和慢速两部分。每一部分都进入一个独立的“线性水库”。
-   **出流**: 每个水库的出流量与当前水库的蓄水量成正比。总出流量是快速和慢速两部分出流之和。

### 主要参数

-   `k_q`: 快速流出流系数，控制快速径流（地表径流）的消退速率。
-   `k_s`: 慢速流出流系数，控制慢速径流（壤中流或基流）的消退速率。

### 使用示例

```python
from hydro_model.routing import SimpleRouting

# 创建一个实例
# 参数代表快速流每日消退80%，慢速流每日消退10%
routing_module = SimpleRouting(k_q=0.8, k_s=0.1)

# 模拟当产流模块产生10mm径流深时的出流
outlet_flow = routing_module.run(inflow=10)

print(f"演算后的出流量为: {outlet_flow:.2f} mm")
```

## 2. `UnitHydrographRouting`

**位置:** `hydro_model/routing.py`

单位线法是水文学中一个非常经典的汇流计算方法。

### 核心思想

-   **单位线 (Unit Hydrograph, UH)**: 单位线是一个流域对于一个单位（如1mm）的有效降雨，在特定时段内（如1天）产生的出流过程线。它像一个“指纹”，反映了流域的综合汇流特性。
-   **卷积**: 对于任意的连续有效降雨过程，其总的出流过程线可以通过将降雨过程与单位线进行**卷积**运算得到。

### 主要参数

-   `uh_ordinates`: 一个列表或Numpy数组，包含了单位线的纵坐标。例如 `[0.1, 0.4, 0.3, 0.15, 0.05]` 代表一场单位降雨将在接下来的5天内，分别产生0.1, 0.4, 0.3...单位的出流。

### 使用示例

**脚本:** `examples/run_uh_example.py`

这个示例清晰地展示了卷积过程。它定义了一个单位线和一系列降雨脉冲，然后计算并绘制出最终叠加形成的流量过程线。运行以下命令来查看：
```bash
python examples/run_uh_example.py
```

## 3. `MuskingumRouting`

**位置:** `hydro_model/routing.py`

马斯京根法是一种常用的河道洪水演进计算方法。与单位线法不同，它更多地用于模拟径流在**单一河段**中的传播过程，而不是整个流域的汇集过程。

### 核心思想

-   **水量平衡**: 模型基于一个简单的水量平衡方程：入流量 - 出流量 = 蓄水量的变化。
-   **槽蓄方程**: 它创造性地提出，河段的蓄水量 `S` 不仅与出流量 `O` 有关，也与入流量 `I` 有关，即 `S = K * [x*I + (1-x)*O]`。
-   **汇流方程**: 基于以上两点，可以推导出一个简单的线性方程来计算当前的出流量 `O_t`。

### 主要参数

-   `K`: 蓄流时间常数 (单位：天)。它反映了洪水波通过该河段所需的大致时间，`K`越大，延迟越明显。
-   `x`: 权重因子 (0-0.5)。它反映了槽蓄作用的权重，`x`越小，洪峰的削减作用越强。

### 使用示例

**脚本:** `examples/run_muskingum_example.py`

这个示例将一个三角形的洪水波输入到一个具有特定`K`和`x`参数的河段中，并绘制出被削减和延迟后的出流过程线。运行以下命令来查看：
```bash
python examples/run_muskingum_example.py
```
