# 第12章：自动化流域划分与参数分区

在准备好基础GIS数据后，我们就可以利用框架提供的工具来执行最激动人心的任务之一：**自动化地划分亚流域并赋予它们物理上有意义的参数分区ID**。

这个过程将原本需要GIS专业软件进行繁琐手动操作的工作，简化为了运行一个Python脚本。

## 核心工具

这个任务的核心是 `whitebox-tools`，一个功能极其强大的地理空间分析库。我们通过`whitebox` Python包来调用它的功能。

## 示例脚本与工作流程

**脚本:** `examples/generate_parameter_zones.py`

这个脚本完整地实现了从原始DEM到最终带有参数分区的子流域矢量文件的全过程。让我们分解它的工作流程：

### 第1步: DEM预处理

这是所有水文分析的起点。
-   **`wbt.fill_depressions()`**: 填洼。DEM数据中常常存在一些低于其周围像素的“洼地”，这会中断水流的计算。此工具将这些洼地填平，确保水流路径的连续性。
-   **`wbt.d8_pointer()`**: 计算D8流量方向。它会为每个像素生成一个值（1, 2, 4, 8, ...），指出水流将流向其8个邻居中的哪一个。这是后续所有分析的基础。

### 第2步: 提取河网与划分亚流域

-   **`wbt.d8_flow_accumulation()`**: 计算流量累积。它会计算每个像素的上游汇流面积（以像素数量计）。流量累积值越大的地方，通常就是河道所在的位置。
-   **`wbt.extract_streams()`**: 基于一个流量累积阈值，从流量累积栅格中提取出河网。
-   **`wbt.subbasins()`**: 这是非常关键的一步。它根据D8流量方向和提取出的河网，为**每一条河段**划分出其对应的**亚流域**。输出是一个栅格文件，其中每个亚流域的像素都被赋予了相同的ID值。

### 第3步: 栅格转矢量

-   **`wbt.raster_to_vector_polygons()`**: 将上一步生成的亚流域栅格，转换为矢量多边形（Shapefile）。这样做的目的是为了方便后续与同样是矢量格式的土地利用、土壤数据进行空间分析。

### 第4步: 空间叠加与参数分区

这是画龙点睛的一步，完全在`geopandas`中完成。
-   **加载数据**: 使用`geopandas`读取亚流域、土地利用和土壤的Shapefile文件。
-   **空间叠加 (`gpd.overlay`)**: 将亚流域图层分别与土地利用图层和土壤图层进行“相交(intersection)”操作。这会产生新的图层，其中包含了被亚流域边界切割后的土地利用和土壤多边形。
-   **计算主导类型**: 对于每一个亚流域，脚本会计算其内部所有土地利用图斑的面积，并找出面积最大的那个作为该亚流域的**主导土地利用类型**。对土壤类型也执行同样的操作。
-   **生成`zone_id`**: 最后，将每个亚流域的主导土地利用类型和主导土壤类型用字符串连接起来（例如，`Forest-Clay`），生成一个独一无二的**参数分区ID**，并将其作为新的属性列添加到亚流域的Shapefile中。

## 运行与验证

您可以亲手运行这个过程：
```bash
# 首先确保您有GIS数据，如果没有可以先运行
# python examples/create_gis_data.py

# 运行主程序
python examples/generate_parameter_zones.py
```
运行成功后，最重要的输出文件是 `examples/results/subbasins_with_zones.shp`。您可以使用任何GIS软件（如QGIS）打开它，查看其属性表，您会发现每个亚流域多边形（Subbasin）都有一个`VALUE`（亚流域ID）和我们刚刚生成的`zone_id`。

您也可以运行`python examples/plot_zones.py`来生成一张可视化的地图，直观地检查分区结果。

## 总结

通过这个强大的工作流，我们成功地将繁琐的GIS操作自动化，并最终得到了可以直接用于指导水文模型参数设置的、带有物理意义的参数分区。在下一章，我们将探讨如何利用这些分区结果。
